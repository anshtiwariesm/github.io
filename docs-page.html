Getting started Last updated: 17-02-2021

Our promise is to help you build and launch your own customized marketplace in 14 days or less.
Launch your own marketplace by following these steps.

Step 1 - Registration

The first step to getting started with eSamudaay is to register with us by sending an email. Our tech team will get back to you with your Application key(TPID). This TPID allows you to access our APIs.

Step 2 - Source Code for mobile applications

We will also provide you with a source code for mobile applications. These are fully functional production-ready mobile applications built in flutter. You will have the right to modify the source code the way you like.
Here are the current production versions of these mobile applications.

  • Merchant App
  • Consumer App
  • Delivery App

Step 3 - Customization Support Frontend

In case you need support for customization on mobile applications, we will connect you with our partner developer agencies.
Our partner developer agencies will help you customize the apps for your need and launch them on the Playstore/Appstore.

Step 4 - Circle Promoter Console

You will get access to a console (CP console) to monitor your marketplace at all times. This console will allow you to be on top of essential metrics like new orders, delivery status, businesses to be onboarded, lead management, transaction reports, etc.
CP console is built using low code technology, thus allows for customization as per your need.

Step 5 - Testing and Support

You will be provided with a testing environment to allow you to test the marketplace the way you like quickly. eSamudaay team will help you with test support. We will provide best practices on configuring your product details like product catalog, choice of businesses, home screen categories, banners, etc.

Step 6 - Launch

The launch is two steps -

  • Publish your apps to Playstore/Appstore with production TPID. You may take support from our partner developer agencies for this.
  • Spread the word by sharing your apps through relevant channels and see your marketplace grow!

Infrastructure

Our goal is to build a technology ecosystem that enables developer entrepreneurs to start, manage and grow their commerce circle by providing them digital tools to promote self-reliance and compete on quality of customer experience with global leaders.

We allow developers to fork/clone from our bitbucket repository as far as our technology infrastructure is concerned. We use flutter/dart, Django framework for APIs, and digital ocean for hosting.

  • We use two environments - Staging and Prod.
    • Staging (1 Droplet): 4 cores, 8GB RAM, 160GB disk
    • Production (2 Droplets): 4 cores, 8GB RAM, 160GB disk
  • DO Spaces stores static data.
  • Our Primary Database is Postgres
    • Hosted Psql 11 on DO
    • Runs on 1 CPU + 2 GB + 25 GB Disk
    • Use Connection pooling to mitigate performance issues due to high CPU usage.
  • Redis
    • To run async and periodic tasks.
    • Handle throttling
    • Redisearch
    • Two different instances per environment

Behind the droplets, we have Dockerized containers that run all the workers and Nginx server, Celery for Periodic tasks, and 2 Async workers - Fast & slow based on priorities.

Installation

Set up the environment by following these 5 steps


Step 1 - Flutter installation
Install the flutter framework by following this link.

Step 2 - Clone the code
Clone the source code from our bitbucket repository. To access our bitbucket repository, follow Getting started.

Step 3 - Add TPID
Once you get the TPID from our team, you have to follow this path and replace third Party Id with assigned one.
Consumer App - esamudaay_consumer/lib/utilities/stringConstants.dart 

													
	/// Replace the value for thirdPartyId here 
	static const thirdPartyId = "YOUR_VALUE_HERE";


Step 4 - Staging and production environment
Switch to staging and prod environment by following this path in the our root directory.
Consumer App - esamudaay_consumer/lib/utilities/stringConstants.dart 

												
	/// Switch environment value here to define the development/production phase. 
	/// for developement purpose, use ENVIRONMENT.DEVELOPMENT
	/// for production, use ENVIRONMENT.PRODUCTION 
	static const ENVIRONMENT environmentVariable = YOUR_VALUE_HERE;


Step 5 - Flutter run
Once you are done with the above steps, you can run and execute the application by following this command.

	flutter run

Directory structure

Once you fork/clone esamudaay_consumer on your repository/local machine, for your smooth onboarding with eSamudaay we recommend you understand the flow of our directory structure.

The structure looks like this.


											
	android/ 
	assets/
		font/ 
		images/
		languages/ 
	ios/ 
	lib/
		modules/ 
		payments/
		redux/ 
		repository/
		reusable_widgets/ 
		utilities/
		main.dart/ 
	test/
	.gitignore/
	README.md/
	pubspec.yaml/
  • fonts - This folder consists of custom fonts, add the tff file of new custom fonts here.
  • images - A seperate folder for images used through out the application e.g. banners, logos, etc.
    For each image, add three resolutions and showing it in the application will be handled by flutter..

  • languages - Consist of all JSON files for supported language translations.

  • main.dart - Entry point of code execution.
  • modules - We use MVC pattern.
    For each folder of the features there are following sub-directories.
    • model - Model file is to define how data is structured
    • views - All flutter UI/UX logic. It has sub-folders - widgets(components) and subviews
    • action - A dart file which consist of all business logic regarding the features
  • reusable_widgets - Consist of common components which are used across the application.
  • utilities - Consists of services and constants
  • redux - We use async redux for state management purpose. Each dart file within state folder consist of state variable for individual features.

Features


Circle

A 'Circle' is a community of shop owners, restaurants, and merchants that serve in a locality.

Circle Search
Provided an option of location-based selection of circles by just turning GPS on..

Trending Circles
It shows the customizable list of circles where a lot of action is going on. We use high-quality, optimized search algorithms to fetch trending circles based on the merchant's activities and demand in the given circle.

Saved Circles
List of saved circles in the user's profile for quick browsing and selection.

Suggested Circles
A list of circles suggested based on the user's current location; will encourage them to explore more!


Home screen

We allow CP to do Top-level customization in their Circle on the home screen.
For example; In Kolkata - Where the locals predominantly consume fishes so CP can create a specific banner in which on tap users will see a list of restaurants/fish sellers in that Circle.
Based on past orders, bookmarks, etc. merchant list can be customized
On the home screen, we allow searching merchants based on the products. For example, if you search for leather shoes, the home screen will list merchants who have leather shoes in their shop.
Show Circle promotional-specific banners to make it more localized. For example, in the Agra circle, CP can put the Taj Mahal in the upper banner.
Connect with store owner directly
We follow the decentralization of the data and hence provide 100% transparency between the end-users.
Business Videos
Where trending businesses can showcase their products.



Product catalog

Upload the images of the shopping list
Supports feature of uploading upto 5 images per order, we use image compression feature for better efficiency.

Business Categories
Based on classes, products can be divided into business categories.

Business Note
A configurable message(offers, store timings, etc.) put up by business for consumers.











Order Placement & Tracking

Payment

Address

Delivery/Pickup

Notes

Quick actions

Order tracking

Rate the business

Customer Support
