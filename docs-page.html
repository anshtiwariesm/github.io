Getting started Last updated: 23-02-2021

We promise to help you build and launch your customized marketplace in 14 days or less.
Launch your marketplace by following these steps.

Step 1 - Registration

The first step to getting started with eSamudaay is to register with us by sending an email. Our tech team will get back to you with your Application key(TPID). This TPID allows you to access our APIs.

Step 2 - Source Code for mobile applications

We will also provide you with a source code for mobile applications. These are fully functional production-ready mobile applications built with flutter. You will have the right to modify the source code the way you like.
Here are the current production versions of these mobile applications.

  • Merchant App
  • Consumer App
  • Delivery App

Step 3 - Customization Support Frontend

In case you need support for customization on mobile applications, we will connect you with our partner developer agencies.
Our partner developer agencies will help you customize the apps for your need and launch them on the Playstore/Appstore.

Step 4 - Circle Promoter Console

You will get access to a console (CP console) to monitor your marketplace at all times. This console will allow you to be on top of essential metrics like new orders, delivery status, businesses to be onboarded, lead management, transaction reports, etc.
CP console is built using a low-code technology, thus allows for customization as per your need.

Step 5 - Testing and Support

You will be provided with a testing environment to allow you to test the marketplace the way you like quickly. eSamudaay team will help you with test support. We will provide best practices on configuring your product details like product catalog, choice of businesses, home screen categories, banners, etc.

Step 6 - Launch

The launch is two steps -

  • Publish your apps to the Playstore/Appstore with a production environment. You may take support from our partner developer agencies for this.
  • Spread the word by sharing your apps through relevant channels and see your marketplace grow!

Infrastructure

Our goal is to build a technology ecosystem that enables developer entrepreneurs to start, manage and grow their commerce circle by providing them digital tools to promote self-reliance and compete on quality of customer experience with global leaders.

We use flutter/dart for front-end, Django framework for backend APIs, and Digital Ocean for hosting and all cloud services.

  • We use two environments - Staging and Prod.
    • Staging (1 Droplet): 4 cores, 8GB RAM, 160GB disk
    • Production (2 Droplets): 4 cores, 8GB RAM, 160GB disk
  • DO Spaces stores static data like images, files, etc.
  • Our Primary Database is Postgres
    • Hosted Psql 11 on DO
    • Runs on 1 CPU + 2 GB + 25 GB Disk
    • Use Connection pooling to mitigate performance issues due to high CPU usage.
  • Redis - A secondary database unit mainly used to
    • Run async and periodic tasks.
    • Handle throttling
    • Redisearch - To handle fast searches.

Behind the droplets, we have Dockerized containers that run all the workers and Nginx server, Celery for Periodic tasks, and 2 Async workers - Fast & slow based on priorities.

Installation

Set up the environment by following these 5 steps


Step 1 - Flutter installation
Install the flutter framework by following this link.

Step 2 - Clone the code
Clone the source code from our bitbucket repository. To access our bitbucket repository, follow Getting started.

Step 3 - Add TPID
Use this command-line argument to set your third-party ID.

	--dart-define=ThirdPartyID="YOUR_VALUE_HERE"

Tip

Set defaultValue for ThirdPartyID argument to avoid passing it with each command.



Step 4 - Staging and production environment
Use this command-line argument to use the production environments.

	--dart-define=isProductionEnvironment=true

By default, staging environment is selected.


Step 5 - Flutter run
Once you are done with the above steps, you can launch the application by running this command.

	flutter run

Consumer App

An SDK to get home delivery of everything from restaurants, Kirana stores, vegetable stores, clothing stores, laundry services, and any local business near you.
Earn by helping your local businesses and support your local economy.

Directory structure

Once you fork/clone esamudaay_consumer on your repository/local machine, we recommend you understand the flow of our directory structure for your smooth onboarding with eSamudaay.

The structure looks like this.


											
	android/ 
	assets/
		font/ 
		images/
		languages/ 
	ios/ 
	lib/
		modules/ 
		payments/
		redux/ 
		repository/
		reusable_widgets/ 
		utilities/
		main.dart/ 
	test/
	.gitignore/
	README.md/
	pubspec.yaml/
  • fonts - This folder consists of custom fonts. Add the tff file of new custom fonts here.
  • images - A seperate folder for assets images used throughout the application, e.g., banners, logos, etc.
    For each image, add three resolutions and flutter itself maps the most closely matched image resolution to device pixel ratio.
  • languages - Consist of all JSON files for supported language translations.
  • main.dart - Entry point of code execution.
  • modules - We use MVC patterns.
    For each folder of the features there are following sub-directories.
    • model - Model file is to define how data is structured
    • views - All flutter UI/UX logic. It has subfolders - widgets(components) and subviews
    • action - A dart file which consist of all business logic regarding the features
  • reusable_widgets - Consist of common components which are used in multiple modules across the application.
  • utilities - Consists of services and constants.
  • redux - We use async redux for state management purposes. Each dart file within the state folder consists of state variables for individual features.

Features


Circle

A 'Circle' is a community of shop owners, restaurants, and merchants that serve in a locality.
  • Location-based selection of circles just by turning GPS on.
  • A list of suggested circles based on the user's current location.
  • Customizable list of circles where a lot of action is going on. We use high-quality, optimized search algorithms to fetch trending circles based on the merchant's activities and demand in the given circle.
  • List of saved circles in the user's profile for quick browsing and selection.









Home screen

  • Developers can do top-level customization in their circle on the home screen. For instance, In Kolkata - Where the locals predominantly consume fishes so the developer can create a specific banner in which on tap users will see a list of restaurants/fish sellers in that Circle.
  • Based on past orders, bookmarks, etc. the merchant list can be customized.
  • Searching merchants based on the products. For instance, to search for leather shoes, the home screen will list merchants who have leather shoes in their shop.
  • Circle promotional-specific banners to make user experience more localized. For instance, in the Agra circle, CP can put the Taj Mahal images in the upper banner.
  • Connect with the store owner directly via call or Whatsapp since out data is decentralized, i.e., 100% transparency between the end-users.
  • Business videos where trending businesses can showcase their products.


Product catalog

  • Products can be categorized into business categories with their respective images as shown in screenshot 1.
  • Each business category can have their sub-categories as shown in screenshot 2.
  • In case of a single business category, a list of products and their respective images will be shown.
  • Allows searching items on the product screen.
  • Support the option of spotlight, where consumers can see top 5 best selling products of that specific business.
  • Option of business Note, a configurable message(offers, store timings, etc.) put up by business for consumers.










Order Placement & Tracking

  • We have 3rd party Razorpay integration, which allows multiple payment options like debit cards, credit cards, net banking, UPI, wallet, etc.
  • Supports multiple addresses along with landmark and configurable address names.
  • Supports delivery or pickup.
  • Supports uploading up to 5 images per order. We use the Image compression feature whenever and wherever needed. Image compression aims to reduce the irrelevance and redundancy of the image data to store or transmit data in an efficient form in all three applications.
  • Reorder, cancel, or send a note to the merchant for an enhanced experience.
  • Supports order tracking to monitor the progress of the shipment.

Packages

  • Themes
  • In-app update services
  • Image services

Personalization

  • Languages - Supports English, Hindi, Tamil, Telugu, Kannada.
  • Themes - Developers can customize themes based on their requirements.
    follow these steps to create new theme -
    • Add a new Value in enum THEME_TYPES. 
      
	enum THEME_TYPES {PRIMARY_THEME ,SECONDARY_THEME, ... }
    • Create a new color class extending AppThemeColors. 
      	
	class DarkThemeColors with AppThemeColors
	{ 
		@override 
		Color get primaryColor => const Colors.blue, 
		.... 
		@override 
		Brightness get brightness => Brightness.dark; 
	}
    • To update theme 
      
	CustomTheme.instanceOf(buildContext).changeTheme(theme_type);
    • To access theme data 
      
	for colors and text styles : 
	color : CustomTheme.of(context).colors.primaryColor; 
	style : CustomTheme.of(context).textStyles.headline6; 
	otherwise 
	appBarTheme : Theme.of(context).appBarTheme.

Merchant App

An SDK to help merchant create their online marketplace.

Directory structure

Once you fork/clone esamudaay_merchant on your repository/local machine, we recommend you understand the flow of our directory structure for your smooth onboarding with eSamudaay.

The structure looks like this.


						
	android/ 
	assets/
		icon/
		locale/ 
	ios/ 
	lib/
		data/ 
		environments/
		utils/ 
		widgets/
		main.dart/ 
	test/
	fonts/
	.gitignore/
	README.md/
	pubspec.yaml/
	pubspec.lock/
	.metadata/
  • fonts - This folder consists of custom fonts. Add the tff file of new custom fonts here.
  • icon - A seperate folder for assets images used throughout the application.
  • locale - Consist of all JSON files for supported language translations.
  • main.dart - Entry point of code execution.
  • data - Contains following sub-directories.
    • model - Model file is to define how data is structured
    • constants - They contains string, URL paths, etc.
    • bloc - Consists multiples dart files of all business logic regarding the features.
      We use BLOC for state management purposes.
  • modules - It is generic name of all feature specific folder and these directories contains view logic.
  • environment - Contains global constants like TPID, production and stagging environment.
  • widgets - Consist of common components which are used across the application.
  • utils - Consists of services and constants

Features


Merchant Profile

  • Merchants have the option to add multiple businesses.
  • Merchants have the option to share store links to different social media platforms.
  • Support option to add business categories, it is a predefined list of categories which merchants can choose based on the store theme/inventory.
  • Customizable store name, store logo, store description, store address, notice, etc.
  • Option to set store open/close and delivery available/pickup.
  • Merchants will have the option to contact the consumers directly in case of offers, item unavailability, delay in delivery, etc.
  • Supports email notification, by turning this on, merchant will get an email notification for every new order or changes the end-user makes.


Order Screen

  • This is an order management dashboard.
  • For every new order, the merchant will get an option to check or reject the order.
  • Once the merchant checks the orders with the store's inventory, they are good to proceed further, and the consumer will get a notification on the cart screen.
  • Once the order is ready, in case of pickup by consumer, click on "mark completed" and receive the payment if consumer opted for COD, whereas in case of delivery, click on "assign."
  • Merchant can cancel the order in case the kitchen full, item out of stock, no delivery person available, store closing time, etc.





Product screen and business videos


  • Integrated master catalog, which contains a list of predefined products with images in the inventory system.
  • A separate section, to view a list of individual items in the master catalog.
  • Apart from master catalog, merchants can add their own products along with images in the inventory.
  • Support the option of spotlight, where merchants can display their top 5 best selling products.
  • A separate section to select "out of stock items" for merchants which will be seen by consumers.
  • Business videos where merchants can showcase and advertise their businesses.




Google reviews

Merchant app lets you collect customer reviews for free and improve your Google ranking & rating.
  • Send SMSes to customers for free, asking them for Google reviews.
  • Reply to all your Google reviews using the merchant app.
  • Create or manage all your Google business listings from the merchant app.
  • Get real-time app notifications as soon as you receive a new review.
  • Merchant app provides you a highly secure & safe app to manage your customer database.

Circle Promoter console

Powerful Dashboard kit for CPs by eSamudaay built in a low-code platform, thus allows customization with ease.

Features

  • All the workflow in the circle is controlled by the CPs in the console.
  • Auto-refresh dashboard i.e. real-time monitoring.
  • CPs can manage and create leads for the merchants in the console.
  • CPs can add multiple merchants with the help of a console.
  • CPs can create multiple circles with the help of a console.
  • CPs can manage and integrate master catalog into merchant's profile.
  • CPs can see list all orders in the circle for each merchant in console.
  • With the help of console CPs can do delivery agent management.
  • Console will help in payment settlements between CP, Merchant and Delivery agents.

Lead management

  • Manage complete lead life cycle
  • Create a single lead or upload in bulk
  • Schedule & track all activities

Seller management

  • Captures business details/documents.
  • Create/upload the business catalog.
  • Get assistance from master catalog(which contains a list of predefined products with images in the inventory system).
  • In console CPs can make the businesses live on apps.
